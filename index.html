Is an interpreted scripting language (code is not precompiled). It is used to add interactive behavior to HTML pages. You can embed it directly into an HTML page, but writing it as a separate JS file helps separate structure from behavior. Unlike server-side scripting languages such as PHP and ASP, JavaScript is primarily used as a client-side scripting language to run on the user's browser and does not require server support. So in the early days programmers preferred JavaScript to reduce the burden on the server, but at the same time brought another issue: security. As servers get stronger, JavaScript continues to thrive because it's cross-platform and easy to use, although today's programmers prefer scripts run on the server side for security. At the same time, some special functions, such as AJAX, must rely on Javascript to support them on the client side. With the development of engines such as V8 and frameworks such as Node.js, with features such as event-driven and asynchronous IO, JavaScript is increasingly being used to write server-side programs.
